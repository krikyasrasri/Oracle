
Steps:

1)Find the OS Version using uname -a 

Move using winscp the Respective Sun/Aix RDA Zip files to any common filesystem like  /tmp and unzip it.

2)Run the following scripts:

 cd /tmp/rda

./rda.sh -cv  --> Loading &Checking the directory .You should get "No issues found".

./rda.sh -S    --> Give the appropriate values(y/n) for the database releted questions w.r.t 
		   Oracle_Home,domain(mastercard.net),Oracle_SID,ASM,RMAN,DATAGAURD,Database Hang Analysis,Oracle VPD,RDBMS Information etc.

	             FOR ALL OTHER questions   like  Collects Oracle 	Communications ASAP 	Information,Oracle Communications Network Discovery 			Information,Hyperion,Siebal,Essbase etc --> Just Hit 'Return' to accept the default (N) 

Finally,Run

./rda.sh -ny -e SQL_LOGIN=/,SQL_SYSDBA=1 -p Rac_Assessment --> 

creates the required output file(Zip file),just copy it using winscp to location V:\Arch Services General\Data Architecture-offshore\RDA and  rename it as RDA.RDA_Servername_<SID>.zip

cd /tmp/rda/output

chmod 755 /output

$ ls -lrt *.zip
-rw-r-----    1 oracle   dba        17325744 Sep  2 08:40 RDA.RDA_cds2ksc40.zip

chmod 755  RDA.RDA_cds2ksc40.zip

Once copied,update status as "completed" in the XLS & go ahead with next DB....

 sqlplus should connect to the database with the userid specified during the
 setup.

 With the same command used so far, start the data collection:

    ./rda.pl [-v]
 or
    ./rda.sh [-v]

 The -v option is optional. It allows to see the collection progression.

 Once it has completed, the results will be packaged for sending it to Oracle
 Support Services, if this file was generated to assist in resolving a Service
 Request.

 Selective data collection is also possible. For more details, the command
 usage can be displayed by specifying the -h option, or complete manual page
 with the -M option.
